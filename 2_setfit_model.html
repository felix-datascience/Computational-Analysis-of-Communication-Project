SetFit Few Shot Classifier¶

This notebook documents the second proposed solution for automatically annotating the YouTube comments dataset. The solution of this notebook is based on SetFit [0], which utilizes large language models under the paradigm of few-shot learning. Contrary to the previous GPT3-based zero-shot learning approach, this approach requires labeled training examples. Still, SetFit can learn from considerably fewer training examples than many other approaches that utilize language models. While the authors of SetFit claim that SetFit is fast to train, the code in this notebook was run with GPU on BWUniCluster 2.0 in order to train the models within a reasonable amount of time.

Train SetFit Classifiers¶

The following function is used to train a SetFit model on predicting one attribute at a time. To finally obtain a full dataset containing annotations for all attributes, multiple models are trained and then used to annotate the unlabeled dataset.

SetFit uses pre-trained sentence transformer models and fine-tunes them with contrastive learning. Contrastive learning is a few-shot learning technique that increases the amount of training data by forming positive and negative pairs of training examples. A positive pair contains examples of the same class while a negative pair contains examples of different classes. The model is then fine-tuned to increase the similarity of embeddings of positive pairs and decrease the similarity of embeddings of negative pairs.

One important group of hyperparameters of the SetFit model is concerned with the sampling strategy that is used to generate positive and negative pairs. We chose to first reduce the training dataset size by sampling examples of all attribute and label combinations. The table below shows the value counts for all attributes in the training dataset.

It can be seen that there are plenty of examples for most attributes and values, but there are also attribute and value combinations with only a few examples (e.g. negative sentiment_overall and positive memory_relate value). We chose to sample 20 positive and negative examples, as this sample size is possible for most attribute and value combinations. In cases where there are fewer than 20 examples, the maximum amount is sampled

Another hyperparameter is num_iterations. This hyperparameter controls the number of positive and negative pairs that are constructed from each example. We set this parameter to 10. As we have 20 positive and negative examples for most attributes, this generates mostly unique pairs. Only in cases where the number of samples per attribute and value is below 10 this sampling strategy leads to duplicates. Assuming, a binary attribute with at least 10 positive and negative examples, the number of training pairs can be calculated as follows:

$(10\ positive\ samples + 10\ negative\ samples) \cdot 40\ unique\ examples = 800\ unique\ pairs$

After training a SetModel the function displays a short evalution on the test set.

In [ ]:
def train_and_evaluate_setfit_model(
    gold_standard_annotations_train,
    gold_standard_annotations_test,
    gold_standard_comments_train,
    gold_standard_comments_test,
    attribute
):
    """Function for training a setfit model and running a short evaluation"""
    
    # select attribute that the model learns to predict and join with comments
    train = gold_standard_annotations_train[[attribute]].join(gold_standard_comments_train[["comment"]])
    test = gold_standard_annotations_test[[attribute]].join(gold_standard_comments_test[["comment"]])
    train = train.dropna()
    test = test.dropna()
    # create huggingface datasets
    train_dataset = Dataset.from_pandas(train.reset_index(drop=True))
    test_dataset = Dataset.from_pandas(test.reset_index(drop=True))
    # sample train dataset
    train_dataset_sample = sample_dataset(train_dataset, label_column=attribute, num_samples=20)
    # number of classes
    n_classes = len(train[attribute].dropna().unique())

    # load pretrained model
    model = SetFitModel.from_pretrained("sentence-transformers/paraphrase-mpnet-base-v2")

    # train model
    args = TrainingArguments(
        batch_size=16,
        num_epochs=4,
        evaluation_strategy="epoch",
        save_strategy="epoch",
        sampling_strategy="num_iterations",
        num_iterations=10,
        load_best_model_at_end=True,
        seed=42
    )
    trainer = Trainer(
        model=model,
        args=args,
        train_dataset=train_dataset_sample,
        eval_dataset=test_dataset,
        metric="accuracy",
        column_mapping={"comment": "text", attribute: "label"}
    )
    trainer.train()

    # make predictions on the test dataset and evaluate model
    preds = model.predict(test_dataset["comment"])

    acc = accuracy_score(test_dataset[attribute], preds)
    f1 = f1_score(test_dataset[attribute], preds, average="binary" if n_classes == 2 else "micro")
    print(f"accuracy: {acc}\nf1-score: {f1}")
    
    cm = confusion_matrix(
        test_dataset[attribute],
        preds,
        labels=[0, 1] if n_classes == 2 else [-1, 0, 1]
    )
    cm_plot = ConfusionMatrixDisplay(cm, display_labels=[0, 1] if n_classes == 2 else [-1, 0, 1])
    cm_plot.plot()
    plt.show()

    return model

Conclusion¶

The predictions of many attributes, especially in the affective response category, are rather unreliable. Still, there are attributes that could be reliably labeled with the trained SetFit models. The SetFit model contains multiple hyperparameters that can be optimized for better results and we tried out different sampling strategies of which all others were worse than the chosen setting. For an extensive hyperparameter optimization the model training would take substantially more time (training and evaluating all models takes roughly 3 hours on GPU), which is why we decided to stick with the chosen hyperparameters.

In comparison, the previous GPT-3-based approach showed more promising results, but due to the lack of reproducibility and the high cost the SetFit models are used to continue with the project and make predictions for the unlabelled dataset.
